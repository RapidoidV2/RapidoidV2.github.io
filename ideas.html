Rapidoid - Ideas for Google Summer of Code 2016:

(Students are welcome to propose and discuss their own ideas on the mailing list, as well)

Information for students

These ideas were contributed by the lead developer of Rapidoid. They are not final nor complete. If you wish to submit a proposal based on these ideas, or completely new idea, it would be great to first contact the developer(s) on the mailing list rapidoid@googlegroups.com and discuss the details further.

1. Develop high-performance HTTP 2 server

Brief explanation:

Rapidoid consists of HTTP 1.1 server and high-level MVC web framework on top of it.

The HTTP 1.1 server in one of the fastest in the world. (View benchmark)

A great step forward would be to implement the HTTP 2 protocol.

To achieve the highest performance, the implementation should be based on the Rapidoid Net module, similar to the existing HTTP 1.1 implementation.

Expected results:

Fully functional and configurable HTTP 2 server.

Appropriate tests and basic documentation for the server.

Knowledge prerequisite:

Object-oriented programming with Java (mandatory)

Network programming with Java (I/O selectors, buffers, etc.)

Understanding of the HTTP 2 protocol

Mentor:

Nikolche Mihajlovski

Contact:

Rapidoid mailing list rapidoid@googlegroups.com

3. Integrate Google Maps in the web framework (create wrapper widget)

Brief explanation:

Rapidoid consists of HTTP 1.1 server and high-level MVC web framework on top of it.

The Rapidoid GUI module has various widgets already included (grid, form, paging etc.).

Design and implement Google Maps widget in the framework, encapsulating the Google Maps API usage details away from the framework users.

The idea is to enable software developers to add Google Maps to the web aplication using simple Java code:

e.g. gmap(41.9840351, 21.4710672).markers(...).direction(...).menu(...)

Expected results:

Fully functional, configurable and extensible Google Maps widget included in the framework.

Appropriate tests and documentation for the widget.

The widget must support the typical Google Maps features: map style, markers, directions etc.

It should be possible to provide data to the widget in both ways: pre-specified upon construction, or asynchronous loading through Ajax.

Knowledge prerequisite:

Object-oriented programming with Java (mandatory)

HTML, CSS, JavaScript, Ajax (mandatory)

Google Maps API (optional)

Mentor:

Nikolche Mihajlovski

Contact:

Rapidoid mailing list rapidoid@googlegroups.com
