Ideas for Google Summer of Code 2015:

(Students are welcome to propose and discuss their own ideas on the mailing list, as well)

Information for students

These ideas were contributed by the lead developer of Rapidoid. They are not final nor complete. If you wish to submit a proposal based on these ideas, or completely new idea, it would be great to first contact the developer(s) on the mailing list rapidoid@googlegroups.com and discuss the details further.

1. Integrate Google Maps in the framework (create wrapper widget)

Brief explanation:

Rapidoid GUI has various widgets already included (grid, form, paging etc.).

Design and implement Google Maps widget in the framework, encapsulating the Google Maps API usage details away from the framework users.

The idea is to enable software developers to add Google Maps to the web aplication using simple Java code:

e.g. gmap(41.9840351, 21.4710672).markers(...).direction(...).menu(...)

Expected results:

Fully functional, configurable and extensible Google Maps widget included in the framework.

Appropriate tests and documentation for the widget.

The widget must support the typical Google Maps features: map style, markers, directions etc.

It should be possible to provide data to the widget in both ways: pre-specified upon construction, or asynchronous loading through Ajax.

Knowledge prerequisite:

Object-oriented programming with Java (mandatory)

HTML, CSS, JavaScript, Ajax (mandatory)

Google Maps API (optional)

Mentor:

Nikolche Mihajlovski

Contact:

Rapidoid mailing list rapidoid@googlegroups.com

3. Integrate AngularJS in the Rapidoid framework

Brief explanation:

Rapidoid GUI is currently based on Twitter Bootstrap and JQuery.

Include AngularJS in the framework, as well.

Add the AngularJS-specific tag attributes to the HTML DSL:

e.g. li(checkbox().ng_model("todo.done")).ng_repeat("todo in todos")

Add support for easy inclusion of arbitrary AngularJS modules, assumming their resources/dependencies are provided by the developer.

Create and document at least one AngularJS-powered widget as a proof-of-concept (similar to grid, form etc).

Expected results:

Fully functional inclusion and integration of the AngularJS framework in the Rapidoid framework.

Extended HTML DSL to reflect all AngularJS-specific tag attributes.

Appropriate documentation for the AngularJS-specific tag attributes in the HTML DSL.

Basic support for configuration AngularJS modules to be included.

Knowledge prerequisite:

Object-oriented programming with Java (mandatory)

HTML, CSS, JavaScript, Ajax (mandatory)

AngularJS (optional, but good-to-have)

Mentor:

Nikolche Mihajlovski

Contact:

Rapidoid mailing list rapidoid@googlegroups.com

4. Other ideas

Brief explanation:

Rapidoid is a complex full-stack project, so here are some draft example ideas covering various layers:

- implement data store for the embedded in-mem database based on Redis or MongoDB or BigTable or DynamoDB etc.

- add UDP protocol support to the low-level network module,

- complete the high-performance WebSocket server using rapidoid-net network protocol sub-framework,

- implement high-performance WebSocket client using rapidoid-net network protocol sub-framework,

- integrate DataTables in the framework (create wrapper widgets, similar to the proposed idea about Google Maps),

- integrate a collection of smaller JQuery-based and Bootstrap-based GUI components in the Rapidoid framework (create wrapper widgets, similar to the proposed idea about Google Maps),

- implement the Raft Consensus Algorithm using rapidoid-net network protocol sub-framework,

- integrate Rapidoid networking sub-framework with the Copycat framework.

Expected results:

What is promised should be delivered.

Knowledge prerequisite:

Object-oriented programming with Java (mandatory)

Any experience related to the proposed idea is a great plus.

Mentor:

Nikolche Mihajlovski

Contact:

Rapidoid mailing list rapidoid@googlegroups.com
