Rapidoid - The No-bullshit Web Framework for Java!

Open source (Apache license). Simple. Secure. Extremely fast!

Community: rapidoid@googlegroups.com
Rapidoid consists of several de-coupled modules/frameworks which can be used separately or together: http-fast, gui, web, fluent, u, and more...


Add rapidoid-web as Maven dependency:
<dependency>
    <groupId>org.rapidoid</groupId>
    <artifactId>rapidoid-web</artifactId>
    <version>5.0.5</version>
</dependency>
1 RESTful service one-liner
On GET /size return the length of the "msg" parameter:
 
On.get("/size").json("msg", msg -> msg.length());
GET /size?msg=abc
3

Running this code will start the Http-Fast server embedded in your program.

By default the server listens on port 8888 and address 0.0.0.0 (matching any IP address).

Navigating to http://localhost:8888/size?msg=abc will return 3.

5 Request wrappers (interceptors)
A wrapper executes before the handler:
 
On.defaultWrap((req, next) -> {
    return next.invoke(result -> "Hey: " + result);
});
 
and provides transformation for the result:
 
On.get("/size").json("s", s -> s.length());
On.get("/upper").json("s", s -> s.toUpperCase());
GET /size?s=abcde
"Hey: 5"
GET /upper?s=abc
"Hey: ABC"
6 Returning a "Not Found" result
Returning a null means NOT FOUND:
 
On.get("/").json(req -> {
    return req.params().size() == 1 ? req.params() : null;
});
GET /
404 Not found!
GET /?a=1
{"a":"1"}
GET /?a=1&b=2
404 Not found!
7 Generic handlers match any request
Generic handlers match any request (in the declaration order):
 
On.req(req -> req.data().isEmpty() ? "Simple: " + req.uri() : null);
 
The next handler is executed if the previous returns NOT FOUND:
 
On.req(req -> U.list(req.verb(), req.uri(), req.data()));
GET /foo
Simple: /foo
POST /hey?name=Rapidoid
BODY: {"cool":true}
[POST, /hey?name=Rapidoid, {name=Rapidoid, cool=true}]
8 Asynchronous request processing
Wait 1 second before returning a response:
 
On.get("/").json(req -> Jobs.schedule(() -> {
 
    req.response().content("OK").done();
 
}, 1, TimeUnit.SECONDS));
GET /
"OK"

Usage:

java -cp <yourapp>.jar com.yourapp.Main [option1 option2 ...]

Example:

java -cp <yourapp>.jar com.yourapp.Main port=9090 address=127.0.0.1 cpus=2 workers=4 nodelay

Configuration options:

Key Value
mode=(dev|production) configure dev or production mode (default: production unless debug mode detected)
secret=<SECRET> configure app-specific secret token for encryption (default: null)
port=<P> listen at port P (default: 8888)
address=<ADDR> listen at address ADDR (default: 0.0.0.0)
threads=<T> start T threads for the job executor service (default: 100)
cpus=<C> optimize for C number of CPUs (default: the actual number of the CPUs)
workers=<W> start W number of I/O workers (default: the configured number of CPUs - cpus options)
nodelay set the TCP_NODELAY flag to disable Nagle's algorithm (default: false)
blockingAccept accept connection in BLOCKING mode (default: false)
bufSizeKB=<SIZE> TCP socket buffer size in KB (default: 16)
25 Full application example (rapidoid-web + rapidoid-fluent modules)
Main.java
public class Main extends GUI {
 
  public static void main(String[] args) {
 
    Conf.args(args);
 
     The application will manage this configuration:
 
    Map<String, String> cfg = U.map("title", "App", "user", "u1");
 
     RESTful service for configuration retrieval:
 
    On.get("/cfg").json(cfg);
 
    PageMenu menu = PageMenu.from(U.map("Home", "/", "Group", "/group"));
 
     Wrap every result into a page:
 
    On.defaultWrap((data, next) -> next.invoke(x -> {
        return page(x).menu(menu).title(cfg.get("title"));
    }));
 
     Create buttons to filter by starting letter:
 
    List<Btn> letters = Flow.chars('a', 'z').map(c -> {
        return btn(c).linkTo("/find?p=" + c);
    }).toList();
 
     Display the configuration entries at the home page:
 
    On.page("/").gui(req -> {
        KVGrid grid = grid(cfg).keyView(k -> a(k).href("/edit?k=" + k));
        return U.list(letters, grid);
    });
 
     Group the configuration entries by the starting letter:
 
    On.page("/group").gui(req -> {
        return Do.map(Do.group(cfg).by((k, v) -> k.charAt(0)))
                 .toList((k, v) -> U.list(h3(k), grid(v)));
    });
 
     Edit configuration entries:
 
    On.page("/edit").gui("k", k -> {
        Btn ok = btn("Update").onClick(() -> Log.info("Updated!"));
        return edit(cfg, k).buttons(ok, btn("Back").linkTo("/"));
    });
 
     Search configuration entries by prefix:
 
    On.page("/find").gui("p", p -> {
        return grid(Find.allOf(cfg).where((k, v) -> k.startsWith(p)));
    });
  }
}
